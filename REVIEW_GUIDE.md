## 🚀 快速开始

### 1️⃣ 小米粒开发新技能

```bash
# 小米粒开发并提交Review
bash /root/.openclaw/workspace/scripts/xiaomi_agent_a.sh
```

**流程**：
1. 创建技能
2. 创建feature分支
3. 提交代码
4. 通知米粒儿Review

---

### 2️⃣ 米粒儿Review技能

```bash
# 米粒儿详细Review
bash /root/.openclaw/workspace/scripts/mili_review_enhanced.sh
```

**流程**：
1. 检查Review请求
2. 代码分析
3. 创建详细Review文档
4. 交互式输入评价（12个问题）
5. 做出批准/拒绝决定
6. 提交到Git
7. 通知小米粒

**需要输入的信息**：
```
1. 代码质量评分（1-5星）
2. 主要优点（用逗号分隔）
3. 需要改进（用逗号分隔）
4. 测试是否通过（y/n）
5. 批准发布？（y/n）
6. 为什么批准/拒绝
7. 关键技术点
8. 潜在风险
9. 短期改进建议
10. 长期改进建议
11. 技术建议
12. 协作建议
```

---

### 3️⃣ 小米粒学习并执行

```bash
# 小米粒学习Review并执行
bash /root/.openclaw/workspace/scripts/xiaomi_review_workflow.sh
```

**流程**：
1. 检查Review结果
2. 读取Review文档
3. 学习技术要点
4. 了解改进建议
5. 根据结果采取行动：
   - **通过**：合并分支 → 发布到ClawHub → 清理分支
   - **拒绝**：查看问题 → 修改代码 → 重新提交

## 📊 Review文档内容

### 包含的信息

1. **基本信息**
   - 技能名称
   - Review时间
   - Review者
   - 版本

2. **Review结果**
   - 批准/拒绝
   - 总体评价（星级）

3. **Review思路**
   - 代码质量评估
   - 功能实现评估
   - 最佳实践评估

4. **技术要点**
   - 关键技术点
   - 风险点

5. **改进建议**
   - 短期改进
   - 长期改进

6. **学习要点**
   - 小米粒做得好的地方
   - 需要改进的地方

7. **给小米粒的建议**
   - 技术建议
   - 协作建议

## 💡 使用技巧

### 米粒儿Review技巧

1. **代码质量评分**：
   - 5星：完美，无改进空间
   - 4星：优秀，有少量改进空间
   - 3星：良好，有明显改进空间
   - 2星：及格，需要重大改进
   - 1星：不及格，需要重写

2. **优点和改进**：
   - 具体、可操作
   - 用逗号分隔
   - 示例："代码结构清晰,文档完整,测试充分"

3. **改进建议**：
   - 短期：下个版本能实现
   - 长期：未来考虑

### 小米粒学习技巧

1. **认真阅读Review**：
   - 理解米粒儿的思路
   - 学习技术要点
   - 应用改进建议

2. **记录学习要点**：
   - 保存到memory文件
   - 应用到未来技能

3. **及时反馈**：
   - 有问题及时沟通
   - 修改后重新提交
