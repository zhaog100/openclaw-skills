# 在线状态同步机制

**创建时间**：2026-03-12 20:03
**状态**：✅ 已建立
**优先级**：🟡 中

---

## 🎯 机制目的

确保双米粒协作时，双方能实时了解对方的在线状态，合理安排沟通时机。

---

## 📊 状态定义

### 🟢 在线（Online）
**定义**：当前活跃，可以即时响应
**响应时间**：< 5分钟
**适用场景**：正常工作时段

### 🟡 忙碌（Busy）
**定义**：正在处理任务，响应可能延迟
**响应时间**：5-30分钟
**适用场景**：深度开发、会议中、紧急任务

### 🔴 离线（Offline）
**定义**：不在工作状态，无法即时响应
**响应时间**：> 30分钟或次日
**适用场景**：下班、休息、外出

### ⏸️ 勿扰（DND）
**定义**：专注时间，仅紧急情况打扰
**响应时间**：仅紧急事项立即响应
**适用场景**：深度思考、重要文档编写

---

## 📱 状态同步方式

### 方式1：Issue评论（主要）

**格式**：
```markdown
📱 状态同步：🟢 在线

**当前任务**：Smart Model v2.0 Review
**预计时长**：1小时
**可响应程度**：高

*状态更新 - 2026-03-12 20:00*
```

### 方式2：`.mili_comm/status.json`（自动）

**格式**：
```json
{
  "xiaomili": {
    "status": "online",
    "task": "Smart Model v2.0 Review",
    "duration": "1h",
    "updated_at": "2026-03-12T20:00:00+08:00"
  },
  "milier": {
    "status": "busy",
    "task": "产品构思",
    "duration": "2h",
    "updated_at": "2026-03-12T19:50:00+08:00"
  }
}
```

### 方式3：QQ状态（紧急）

**使用场景**：
- 🔴 离线 → 在线（重要通知）
- 🟡 忙碌 → 在线（任务完成）
- ⏸️ 勿扰 → 在线（专注结束）

---

## 📊 状态变更规则

### 自动变更
- 🟢 在线 → 🟡 忙碌：30分钟无操作
- 🟡 忙碌 → 🔴 离线：2小时无操作
- 任何状态 → 🟢 在线：有新操作

### 手动变更
- 开始深度任务：→ 🟡 忙碌 或 ⏸️ 勿扰
- 任务完成：→ 🟢 在线
- 下班休息：→ 🔴 离线

---

## 📝 状态同步频率

**正常情况**：
- 每小时自动同步一次（通过Git提交）
- 状态变更时立即同步

**特殊情况**：
- 紧急任务开始/结束：立即同步
- 长时间离线：提前告知
