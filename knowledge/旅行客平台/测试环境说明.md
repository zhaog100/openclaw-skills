## 🎯 测试重点

### 核心业务流程

**订单全流程：**
```
创建订单 → 价格计算 → 议价审批 → 订单拆分 → 派单 → 执行 → 完成
```

**验证点：**
1. 价格计算准确性
2. 拆单逻辑正确性
3. 派单流程完整性
4. 状态流转正确性

### 价格体系验证

**三层模型测试：**
1. **费用因子** - 基础费用配置正确
2. **价格规则** - 规则命中准确
3. **规则分类** - 权限管理有效

**验证点：**
- 规则唯一性（必须命中且仅命中1条）
- 价格可追溯（点击费用查看来源）
- 参数回退机制（缺失参数使用默认值）

### 订单拆分逻辑

**拆单维度：**
1. 按行程天数
2. 按车辆
3. 按司机数量

**验证点：**
- 拆单结果正确
- 状态联动正确
- 异常处理合理
