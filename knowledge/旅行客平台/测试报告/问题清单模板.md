## 🎯 BUG等级定义

| 等级 | 定义 | 影响 | 处理时限 |
|-----|------|------|---------|
| **致命** | 系统崩溃、数据丢失、安全漏洞 | 阻塞主流程 | 24小时内 |
| **严重** | 核心功能异常、影响业务 | 影响主要功能 | 3天内 |
| **一般** | 次要功能异常、体验问题 | 影响用户体验 | 1周内 |
| **轻微** | UI问题、文案错误 | 不影响功能 | 迭代时处理 |

---

## 🏷️ BUG类型分类

| 类型 | 说明 | 示例 |
|-----|------|------|
| **功能缺陷** | 功能不符合预期 | 提交失败、数据错误 |
| **UI问题** | 界面显示异常 | 布局错乱、样式问题 |
| **性能问题** | 响应慢、卡顿 | 加载超时、内存泄漏 |
| **安全问题** | 安全漏洞 | 权限绕过、SQL注入 |
| **兼容问题** | 兼容性问题 | 浏览器不兼容、系统适配 |
| **数据问题** | 数据异常 | 数据丢失、数据不一致 |

---

## 📌 优先级定义

| 优先级 | 定义 | 处理顺序 |
|-------|------|---------|
| **P0** | 最高优先级，阻塞发布 | 立即处理 |
| **P1** | 高优先级，影响主流程 | 本迭代完成 |
| **P2** | 中优先级，影响次流程 | 下迭代完成 |
| **P3** | 低优先级，优化类问题 | 择机处理 |

---

## 📊 状态流转

```
新建 → 确认 → 处理中 → 已解决 → 验证通过 → 已关闭
         ↓         ↓         ↓
       拒绝     挂起     验证失败
                            ↓
                         重新打开
```

| 状态 | 说明 |
|-----|------|
| **新建** | 问题刚提交，待确认 |
| **确认** | 已确认是Bug，待分配 |
| **处理中** | 开发正在修复 |
| **已解决** | 开发已完成修复 |
| **验证通过** | 测试验证通过 |
| **已关闭** | 问题完全解决 |
| **拒绝** | 不是Bug或不予处理 |
| **挂起** | 暂时无法处理 |
| **重新打开** | 验证失败，重新修复 |

## 💡 使用建议

### 1. 问题提交规范

**标题格式**：`[模块] 简要描述`
- 示例：`[订单管理] 创建订单时行程天数未自动填充`

**描述结构**：
```
【环境】设备、系统、浏览器
【步骤】复现步骤
【预期】期望结果
【实际】实际结果
【附件】截图、录屏
```

### 2. 问题编号规则

格式：`BUG` + `年月日` + `3位数字`
- 示例：`BUG-20260304-001`

### 3. 附件管理

```
screenshots/
├── bug-001/           # Bug编号
│   ├── before.png     # 问题截图
│   ├── after.png      # 修复后截图
│   └── video.mp4      # 录屏
```

### 4. 日报/周报统计

**统计维度**：
- 新增Bug数
- 已解决Bug数
- 遗留Bug数（按等级）
- Bug修复率

---

## 🎯 质量指标

| 指标 | 计算公式 | 目标值 |
|-----|---------|--------|
| **Bug修复率** | 已解决 / 总数 × 100% | ≥ 95% |
| **Bug重开率** | 重开数 / 总数 × 100% | ≤ 5% |
| **P0/P1解决率** | P0/P1已解决 / P0/P1总数 × 100% | 100% |
| **Bug密度** | Bug数 / 功能点数 | ≤ 0.5 |
