# 旅行客平台 - 系统功能结构

## 🏗️ 系统架构

### 核心定位
**旅游用车综合服务平台**

### 四端设计
1. **管理端** - 规则配置、订单管理、用户管理、统计看板
2. **旅行社Web端** - 线路查看、订单下发、结算管理
3. **司机端（移动）** - 接单、打卡、紧急求助
4. **用户端** - 查看行程

### 技术栈
- **前端框架**：Vben Admin + Vue3 + Vite
- **架构模式**：SPA（单页应用）
- **状态管理**：Vuex/Pinia
- **UI组件**：Ant Design Vue

---

## 🔄 核心业务流程

```
需求创建 → 价格试算 → 定价审批 → 订单拆分 → 派单 → 异常处理 → 行程执行 → 完成
```

### 流程节点详解

**1. 需求创建**
- 旅行社提交用车需求
- 填写行程、人数、服务等级
- 选择计价模式（自动/手动）

**2. 价格试算**
- 系统自动匹配价格规则
- 计算基础费用+附加费用
- 生成报价单

**3. 定价审批**
- 议价流程（如需要）
- 审批通过/驳回
- 最终价格确认

**4. 订单拆分**
- 按行程天数拆分
- 按车辆拆分
- 按司机数量拆分
- 生成子订单

**5. 派单**
- 系统轮询（15分钟周期）
- 指定司机（强派）
- 司机接单确认

**6. 异常处理**
- 订单变更
- 司机调换
- 紧急求助

**7. 行程执行**
- 司机打卡
- 行程跟踪
- 服务完成

**8. 完成**
- 订单结算
- 评价反馈
- 数据归档

## 💰 价格三层模型

```
最上层：规则分类（管理和权限）
    ↓
中间层：价格规则（组合和计算）
    ↓
最底层：费用因子（定义费用本身）
```

### 设计原则
- 新增客户不影响已有规则
- 新增费用不影响已有规则
- 规则可追溯（点击费用查看来源）

### 费用因子类型

**基础费用：**
- 车辆基础费
- 司机基础费
- 导游费

**附加费用：**
- 旺季补贴
- 节假日补贴
- 加班费
- 过路费
- 停车费

**参数配置：**
- 主体（旅行社/司导）
- 城市
- 星级（1-5星+超五星）
- 季节（旺季/淡季）

## 🎯 四大端功能模块

### 1️⃣ 管理端

#### 用户管理
**司导用户管理**
- 注册流程
- 人工审核
- 星级评定（1-5星+超五星）
- 审核状态：待审核/已通过/已驳回
- 评分维度：服务态度、驾驶技术、服务规范

**旅行社用户管理**
- 四步注册流程
  1. 基本信息
  2. 资质材料
  3. 账号密码
  4. 审核通知
- 审核周期：1-3个工作日
- 多端登录限制：最多2台设备

#### 规则配置
**价格规则管理**
- 规则命中逻辑：必须命中且仅命中1条规则
- 优先级：系统根据条件复杂度自动生成
- 规则启用/停用

**费用管理**
- 费用因子配置
- 参数配置（主体/城市/星级/季节）
- 默认值设置

**时间配置**
- 旺季时间配置
- 其他时间默认淡季
- 对历史订单不生效

#### 业务管理
**订单管理**
- 订单状态流转
  - 议价中
  - 已定价待派单
  - 派单中
  - 已完成
  - 已取消
- 智能拆单（主订单→N个子订单）
- 派单模式
  - 系统轮询（15分钟）
  - 指定司机（强派）

**线路管理**
- 线路信息维护
- 司导信息查看
- 线路审核

**产品管理**
- Tab分类
  - 全部产品
  - 下游产品
  - 适配产品
- 状态管理
  - 已上架
  - 已下架
  - 审核中
  - 已通过
  - 草稿

#### 财务管理
**旅行社结算**
- 结算周期管理
- 佣金比例：固定10%
- 状态：待结算/已结算

**司导报销**
- 审批流程：待审批/已结算
- 费用证明：需提供支付截图或照片

#### 应急处理
**紧急求助管理**
- 严重程度
  - 紧急（红）
  - 严重（橙）
  - 一般（蓝）
- 状态：待处理/已处理/已驳回

**申诉处理**
- 纠纷事实核查
- 责任判定
  - 司机责任
  - 游客责任
- 处理结果：通过/不通过

#### 统计看板
- **核心数据**
  - 订单总数
  - 销售金额
  - 司导人数
  - 旅行社数量
- **订单状态分布**（环形图）
- **异常订单列表**
- **订单销售趋势**（双轴折线柱状图）

## 🔧 技术特性

- **框架**：Vue3 + Vite

- **路由**：Vue Router
- **HTTP**：Axios

- **SPA**：单页应用
- **响应式设计**：支持多端适配
- **组件化**：模块化开发
- **懒加载**：按需加载

用户操作 → 组件事件 → Pinia Store → API调用 → 后端处理 → 数据更新 → 页面渲染
