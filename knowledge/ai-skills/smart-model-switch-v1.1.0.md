## 🎯 核心功能

### 双策略系统 ⭐⭐⭐⭐⭐

**策略1：消息复杂度驱动（快速响应）**
- 长度评分（30%）：<50字(1分) <200字(2分) <500字(3分)
- 关键词评分（40%）：简单(0分) 中等(2分) 复杂(3分)
- 代码检测（20%）：无代码(0分) 少量(2分) 大量(3分)
- 视觉检测（10%）：无视觉(0分) 有视觉(3分)

**策略2：上下文监控驱动（长期保护）**
- 连续2次超85%阈值 → 自动切换Kimi 256k
- 冷却期10分钟 → 防止频繁切换
- 通知间隔5分钟 → 避免重复提醒

### AI主动检测机制 ⭐⭐⭐⭐⭐（v1.1.0核心）

**每次回复前自动检查：**
```bash
scripts/integrate-check.sh
```

**执行逻辑：**
1. 检查上下文使用率
2. 判断是否超过阈值（85%）
3. 连续2次超过阈值 → 输出提醒
4. 无问题 → 静默返回（无输出）

**防骚扰机制：**
- 冷却期：切换后10分钟内不再检测
- 通知间隔：5分钟内不重复提醒
- 连续性要求：必须连续2次超过阈值
- 静默模式：无问题时不输出

### 6种智能模型

| 模型 | 触发条件 | 适用场景 |
|------|---------|---------|
| Flash | 评分0-3分 | 简单问答、快速查询 |
| Main | 评分4-6分 | 常规对话、分析任务 |
| Coding | 检测代码 | 代码生成、审查 |
| Vision | 检测视觉 | 图片分析、截图解读 |
| Complex | 评分8-10分 | 深度分析、架构设计 |
| Long-Context | 上下文≥85% | 长对话、超长文档 |

## 🚀 使用方法

### 安装
```bash
clawhub install smart-model-switch
```

### AI集成（强制）
```bash
# 每次回复前执行
~/.openclaw/workspace/skills/smart-model-switch/scripts/integrate-check.sh

# 无输出 = 检查通过，继续回复
# 有输出 = 需要提醒，先输出提醒再回复
```

### 手动使用
```bash
# 分析消息复杂度
node scripts/analyze-complexity.js "你的消息"

# 智能切换模型
scripts/smart-switch.sh "你的消息"

# 手动切换模型
scripts/switch-model.sh bailian/kimi-k2.5
```
