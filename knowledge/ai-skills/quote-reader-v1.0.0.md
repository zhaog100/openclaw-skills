## 🎯 核心功能

### 智能引用识别 ⭐⭐⭐⭐

**多平台支持（4个平台）：**
- **飞书**：`[message_id:xxx]`、`[quote:xxx]`
- **QQ**：`[reply:xxx]`、`[quote:xxx]`
- **Telegram**：`reply_to_message_id:xxx`
- **微信**：引用消息结构

**通用引用格式：**
- 引用：xxx
- 回复：xxx
- 前文：xxx
- 上文：xxx

### 历史检索 ⭐⭐⭐⭐

**会话历史搜索：**
- 根据消息ID定位引用消息
- 提取上下文（前后各2条）
- 智能关联相关对话

**上下文提取：**
```json
{
  "before": ["前一条消息", "前两条消息"],
  "after": ["后一条消息", "后两条消息"]
}
```

### 智能理解 ⭐⭐⭐⭐

**6种引用意图：**

| 意图 | 关键词 | AI行为 |
|------|--------|--------|
| clarify（澄清） | 什么、是指、意思 | 解释引用内容 |
| supplement（补充） | 补充、还有、另外 | 基于引用扩展 |
| refute（反驳） | 不对、其实、应该是 | 修正引用内容 |
| deepen（深入） | 继续、展开、深入 | 深入分析引用 |
| relate（关联） | 关联、关系、联系 | 关联历史对话 |
| example（举例） | 例子、示例、举个 | 举例说明 |

### AI友好集成 ⭐⭐⭐⭐⭐

**静默模式：**
```bash
scripts/integrate-quote.sh "$USER_MESSAGE"

# 无引用 = 无输出（静默）
# 有引用 = 输出引用信息
```

**友好输出格式：**
```
📋 【引用检测】

引用消息：QMD向量生成需要CUDA支持...
引用意图：clarify
建议回复：解释引用内容的具体含义

上下文：
  user: QMD怎么用？
  assistant: QMD是向量搜索工具...
```

## 🚀 使用方法

### 安装
```bash
clawhub install quote-reader
```

### AI集成（强制）
```bash
# 每次收到用户消息时执行
~/.openclaw/workspace/skills/quote-reader/scripts/integrate-quote.sh "$USER_MESSAGE"

# 无输出 = 无引用，正常处理
# 有输出 = 引用信息，结合引用回答
```

### 手动使用
```bash
# 检测引用
scripts/detect-quote.sh "用户消息"

# 提取引用
scripts/extract-quote.sh "引用信息JSON"

# 完整测试
scripts/test-quote.sh
```
