## 🧠 记忆系统（Memory System）

### 核心组成

#### 1. **MEMORY.md** - 长期记忆（精心维护）
- **定位**：精心提炼的精华，像人类的长时记忆
- **内容**：
  - 关于用户（官家偏好、工作风格）
  - 关键决策（CPU模式、知识库路径、工具调用风格）
  - 核心教训（VMware限制、Token浪费、缺少个性）
  - 核心洞察（API限流处理、上下文超限策略、ClawHub发布流程）
  - 重要里程碑（系统优化、技能完成、问题修复）
- **大小**：~15KB（保持精简）
- **更新频率**：
  - 实时：上下文达到85%自动更新
  - 定时：每周日23:00深度精炼

#### 2. **memory/YYYY-MM-DD.md** - 每日日志（原始记录）
- **定位**：当天发生的原始记录，像人类的日记
- **内容**：
  - 重要事件
  - 系统状态
  - 待办事项
  - 观察记录
- **文件数量**：46个文件（从2026-02-14开始）
- **更新频率**：实时（发生即记录）
- **用途**：
  - 临时记录（不重要的记在这里）
  - 周日精炼时从中提取精华到MEMORY.md

### 更新机制

#### 机制1：**上下文传递**（实时）
- **触发**：上下文使用率达到85%
- **执行**：
  1. 自动保存当前会话到MEMORY.md
  2. 自动创建daily log（如果不存在）
  3. 保存重要信息到daily log
  4. 创建新会话并加载记忆
- **目的**：防止会话超限，保持连续性
- **监控**：每小时检查（context-monitor.sh）

#### 机制2：**定时维护**（每周日）
- **触发**：每周日23:00
- **执行**：
  1. 回顾最近7天的daily logs
  2. 识别重要事件、决策、教训
  3. 提炼精华到MEMORY.md
  4. 移除过时信息
  5. 保持MEMORY.md < 15KB
- **目的**：长期记忆优化，防止MEMORY.md膨胀
- **执行者**：AI（isolated会话）

### 记忆哲学

```
短期记忆 → daily logs（临时存储）
    ↓（周日精炼）
长期记忆 → MEMORY.md（精华保存）
```

**原则**：
- 重要的记MEMORY.md
- 临时记daily logs
- 定期精炼，保持简洁

## 📚 知识库系统（Knowledge Base）

### 核心组成

#### 1. **knowledge/** 目录 - 结构化知识
- **定位**：专业知识库，支持项目管理和测试
- **内容分类**：
  - project-management/（项目管理）
  - software-testing/（软件测试）
  - content-creation/（内容创作）
  - ai-system-design/（AI系统设计）
  - automation/（自动化）
  - system-management/（系统管理）
- **文件数量**：26个文件
- **格式**：Markdown（结构化、易搜索）

#### 2. **QMD索引** - 智能检索系统
- **索引大小**：4.2 MB
- **文档总数**：72个文件
- **向量数量**：205个
- **Collections**：
  - knowledge（26个文件）
  - memory（46个文件）
- **更新频率**：每天2次（12:00 + 23:50）

### 检索方式

#### 方式1：**QMD精准检索**（推荐）
```bash
# 搜索知识库
qmd search "关键词" -c knowledge

# 搜索记忆
qmd search "关键词" -c memory

# 混合搜索（精度93%）
qmd query "如何平衡项目质量和进度" -c knowledge
```

#### 方式2：**memory_search**（MEMORY.md专用）
```bash
# 搜索MEMORY.md
memory_search "关键词"

# 获取片段
memory_get path/to/file.md --from 10 --lines 20
```

### Token节省策略

**传统方式**（浪费）：
```
读取整个MEMORY.md = 2000+ tokens
读取所有知识库 = 10000+ tokens
```

**QMD方式**（高效）：
```
QMD搜索 = 50 tokens
片段读取 = 100 tokens
总计 = 150 tokens
节省 = 92.5%
```

### 更新机制

#### 机制1：**自动维护**（每天2次）
- **时间**：12:00 + 23:50
- **执行**：
  1. 运行 `qmd update`（更新索引）
  2. 新文件自动索引
  3. 更新文件自动重新索引
- **目的**：保持索引最新

#### 机制2：**手动更新**（需要时）
```bash
# 更新索引
qmd update

# 重新生成向量
qmd embed -f
```

### 知识库哲学

```
知识 → knowledge/（结构化存储）
    ↓（QMD索引）
快速检索 → Token节省（92.5%）
```

**原则**：
- 知识要结构化
- 检索要精准
- 避免全量读取

## 🔄 Git库系统（Version Control）

### 核心组成

#### 1. **Git仓库** - 版本控制
- **位置**：/root/.openclaw/workspace
- **作用**：
  - 跟踪所有文件变更
  - 保存版本历史
  - 支持回滚
  - 防止数据丢失

#### 2. **提交历史**
- **最近提交**：
  - 记录技能合并决策
  - 合并agent-browser到playwright-scraper
  - 自动维护（记忆更新、知识库同步）
  - 系统升级总结
- **提交频率**：每天2次（12:00 + 23:50）

### 更新机制

#### 机制1：**自动维护**（每天2次）
- **时间**：12:00 + 23:50
- **执行**：
  1. git add -A（添加所有变更）
  2. git commit -m "自动维护: YYYY-MM-DD - 描述"
  3. 自动提交到本地仓库
- **目的**：自动备份，防止数据丢失

#### 机制2：**手动提交**（需要时）
```bash
# 添加文件
git add 文件名

# 提交
git commit -m "提交说明"

# 查看状态
git status
```

### Git哲学

```
变更 → 自动/手动提交
    ↓
版本历史 → 可回滚、可追溯
```

**原则**：
- 频繁提交，小步快跑
- 自动化备份
- 关键操作立即提交
