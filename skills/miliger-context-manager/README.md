# Context Manager v6.0.0

> 全功能整合版 - 三级预警 + 智能清理 + 预测监控

## 🎯 核心特性

### ⭐ v6.0.0 新特性（2026-03-07）
- **三级预警系统**：70%/80%/90%分级预警（动态阈值）
- **智能清理策略**：light/medium/heavy三级清理
- **预测性监控**：计算对话增长速度，提前1小时预警
- **动态阈值**：根据活跃度自动调整（LOW/MEDIUM/HIGH）
- **压缩算法**：对话历史压缩 + 工具历史压缩 + 去重
- **会话时长监控**：2小时预警，4小时严重

### ✅ 核心功能
- **三级预警系统**：
  - 轻度预警（70%/80%/90%）：根据活跃度动态调整
  - 重量预警（80%）：建议优化，飞书通知
  - 严重预警（90%）：立即行动，QQ + 飞书通知

- **智能清理策略**：
  - 轻度清理：临时文件清理（<5秒）
  - 中度清理：活动历史压缩（<10秒）
  - 重度清理：完全重置（<15秒）
  - 自动触发：根据预警级别自动清理

- **预测性监控**：
  - 活动趋势分析（INCREASING/STABLE/DECREASING）
  - 预测超限时间（1-2小时提前预警）
  - 会话时长监控（2小时预警，4小时严重）
  - 工具调用监控（30次/小时预警，50次/小时严重）

- **动态阈值**：
  - LOW活跃：90%触发
  - MEDIUM活跃：80%触发
  - HIGH活跃：70%触发

```

### 使用
**完全自动 - 只需正常聊天**

1. 继续对话（监控在后台智能运行）
2. 检测到高频活动 → 提醒
3. 检测到错误 → 立即通知
4. 用户根据建议决定是否 /new

## 📊 工作原理

```
监控启动（每5分钟）
  ↓
检测用户活动
  ├─ 无活动 → 跳过（节省token）
  └─ 有活动 → 进入第一层
       ↓
第一层：轻量检查（5分钟）
  ├─ 正常 → 结束
  └─ 异常（>10次）→ 进入第二层
       ↓
第二层：详细分析（1小时）
  ├─ 30次 → 提醒
  ├─ 50次 → 预警
  └─ 错误 → 立即通知
```

## 💡 设计思路

### 基于Hazel_OC的token优化经验
- **两阶段执行**：先用轻量判断，再决定是否深入
- **模型分层**：简单任务快速处理，复杂任务详细分析
- **频率调整**：不是所有情况都需要高频监控

### 核心优势
1. **智能节省**：无活动时跳过检查（节省90%+ token）
2. **高效分层**：轻量检查先行（减少不必要分析）
3. **预测主动**：不是等错误，而是预测问题
4. **用户友好**：简洁建议，不是技术错误
